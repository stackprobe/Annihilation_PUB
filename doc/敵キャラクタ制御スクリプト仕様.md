# KEnemy Script Language Specification

## 概要

KEnemy は、Annihilation における敵キャラクタの動作を記述するためのスクリプト言語である。

主に以下を制御する。

* 敵の移動
* 敵の画像
* 敵の当たり判定
* 敵の体力
* 敵・弾の生成

KEnemy は手続き型言語であり、以下をサポートする。

* 逐次実行
* 条件分岐
* 繰り返し
* 関数呼び出し
* 変数
* 数式

---

## スクリプトファイル

### 配置場所

```text
C:\Dev\Annihilation\Game\Storage\KEnemy
````

### ファイル名

```text
<関数名>.txt
```

例：

```text
ザコ敵.txt
放射弾.txt
ボス第一形態.txt
```

1ファイルが1関数に対応する。

## 文字コード

UTF-8

改行コードは CRLF / LF のどちらでもよい。

---

# コメント

`#` で始まる行はコメント行として扱われる。

コメント行は実行されない。

例：

```text
# これはコメント

画像 敵小型

# 体力設定
HP = 10
````

コメントは行頭にのみ記述可能。

現時点では、行末コメントはサポートしない。

以下は未定義動作とする。

```text
HP = 10 # コメント この書き方は禁止！
```

---

## 実行モデル

KEnemy スクリプトは、上から順番に逐次実行される。

以下の命令は時間を消費する。

* `移動相対`
* `移動絶対`
* `移動角度`
* `待つ`

これらの命令は、完了するまで次の行を実行しない。

例：

```text
移動絶対 700 200 60
待つ 30
```

上記は：

1. 60フレーム掛けて移動
2. 30フレーム待機

の順で実行される。

## KEnemy_ScriptRunner

KEnemy スクリプトは、プログラム側から `KEnemy_ScriptRunner` によって実行される。

```csharp
new KEnemy_ScriptRunner("<関数名>", 初期X, 初期Y, [引数1, 引数2, 引数3...] );
```

例：

```csharp
new KEnemy_ScriptRunner("ザコ敵", 960, 180);

new KEnemy_ScriptRunner("中ボス", 100, 200, 180.0, 10.0, 333);
```

### コンストラクタ引数

| 引数      | 意味      |
| ------- | ------- |
| `<関数名>` | 実行する関数名 |
| `初期X`   | 生成時X座標  |
| `初期Y`   | 生成時Y座標  |
| `引数?`   | 関数に渡す引数群 |

### スクリプトファイル解決

`<関数名>` は以下のファイルへ解決される。

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\<関数名>.txt
```

例：

```csharp
new KEnemy_ScriptRunner("ザコ敵", 960, 180);
```

↓

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\ザコ敵.txt
```

## 変数

### 型

すべての変数は `double` 型である。

整数・浮動小数の区別は存在しない。

### 変数宣言

不要。

初回代入時に自動生成される。

例：

```text
I = 0
ANGLE = 90
```

### 変数名

以下の正規表現に一致する必要がある。

```regex
[A-Z][A-Z0-9]*
```

例：

```text
X
Y
HP
I
ANGLE
TARGETX
P1
P2
```

小文字は使用できない。

---

## 標準変数

各敵キャラクタは以下の変数を持つ。

| 変数名  | 意味    |
| ---- | ----- |
| `X`  | 現在X座標 |
| `Y`  | 現在Y座標 |
| `HP` | 現在体力  |

### X, Y

敵生成時に設定される。

座標系：

* バトルフィールド左上が `(0, 0)`
* X増加方向が右
* Y増加方向が下
* バトルフィールドのサイズは (456 x 532)

### HP

初期値は `1`。

敵弾など、プレイヤーが破壊できない敵は 0 にすること。

例：

```text
HP = 10
```

## 関数

### 関数定義

関数定義はファイル名によって行われる。

```text
放射弾.txt
```

↓

```text
放射弾
```

という関数になる。

ファイル内に関数名や引数名を宣言する構文は存在しない。

## 関数呼び出し

### 呼び出し形式

```text
<関数名> <引数1> <引数2> ...
```

例：

```text
放射弾 5 180
```

---

### 関数呼び出し時の変数

関数へ入る際、呼び出し前の通常変数は退避される。

呼び出された関数内で参照できる変数は、以下のみである。

* `X`
* `Y`
* `HP`
* `P1`, `P2`, `P3`, ...（引数が存在する場合）

`X`, `Y`, `HP` は特殊な変数であり、常にどの関数からでも参照・変更できる。

関数を終了すると、呼び出し前に退避された通常変数が復帰する。

関数内で作成・変更された通常変数は、関数終了時に忘却される。

---

## 関数の引数

関数呼び出し時に渡された引数は、呼び出された関数側で以下の変数に順番に格納される。

| 引数   | 格納先             |
| ---- | --------------- |
| 第1引数 | `P1`            |
| 第2引数 | `P2`            |
| 第3引数 | `P3`            |
| 第4引数 | `P4`            |
| ...  | ... |

例：

呼び出し側：

```text
放射弾 5 180
```

`放射弾.txt` 側：

```text
COUNT = P1
SPEED = P2
```

つまり、関数側で任意の名前にしたい場合は、冒頭で代入して受け取る。

```text
COUNT = P1
SPEED = P2
ANGLE_OFFSET = P3
```

### KCalc

`KCalc` は、`KScripts` 名前空間下のスクリプト用計算処理である。

唯一の公開メソッドは次のとおりである。

```csharp
public static void Run(KVariables variables, string[] arguments)
```

#### 引数

`variables`

* 変数名と値を管理する `KVariables` である。
* 変数の参照には `GetValue` を使用する。
* 計算結果は `SetValue` により左辺の変数名へ代入される。
* 未定義変数を参照した場合、その値は `KVariables` によって `0.0` として解決される。

`arguments`

* 計算式をトークン分割した配列である。
* 例: `A = B + 123` は次のように渡される。

```csharp
new string[] { "A", "=", "B", "+", "123" }
```

#### 式の形式

* 1つ目のトークンは代入先の変数名である。
* 2つ目のトークンは必ず `=` である。
* `=` は2つ目のトークン以外には出現しない。
* 3つ目以降が右辺の式である。

#### 対応する値

* 変数名
* 数値リテラル

数値リテラルの解釈には `double.Parse` を使用する。

#### 対応する演算子

* `+`
* `-`
* `*`
* `/`
* `%`

#### 優先順位

* `(` と `)` による優先順位指定に対応する。
* `*`, `/`, `%` は `+`, `-` より優先される。
* 同じ優先順位の演算子は左から順に評価される。

#### 例

```text
A = B + 123
```

`B` が `10` の場合、`A` は `133` になる。

```text
A = ( B + 2 ) * 3
```

`B` が `10` の場合、`A` は `36` になる。

```text
A = B % 4
```

`B` が `10` の場合、`A` は `2` になる。

## 制御構文

### もし

```text
もし 条件

    ...

もし終了
```

例：

```text
もし HP <= 0

    待つ 60

もし終了
```

### 繰り返し

```text
繰り返し 条件

    ...

繰り返し終了
```

条件が真の間、繰り返す。

例：

```text
I = 0

繰り返し I < 5

    I = I + 1

繰り返し終了
```

## プリミティブ関数

以下の関数はプログラム側で実装される。

---

### 発射

```text
発射 <関数名> <引数1> <引数2> ...
```

敵キャラクタを生成する。

生成された敵は、指定された関数を実行する。

生成位置は呼び出し元の敵の現在位置となる。

例：

```text
発射 弾 90 180
```

内部イメージ：

```csharp
new KEnemy_ScriptRunner("弾", X, Y);
```

このとき、`弾.txt` 側では以下のように受け取る。

```text
R = P1
SPEED = P2
```

---

### 移動相対

```text
移動相対 X Y F
```

現在位置から相対的に `(X, Y)` だけ、`F` フレーム掛けて移動する。

例：

```text
移動相対 100 0 60
```

右へ100pxを60フレーム掛けて移動。

---

### 移動絶対

```text
移動絶対 X Y F
```

絶対座標 `(X, Y)` へ `F` フレーム掛けて移動する。

例：

```text
移動絶対 640 200 120
```

---

### 移動角度

```text
移動角度 R D F
```

角度 `R` 方向へ、距離 `D` を `F` フレーム掛けて移動する。

角度仕様：

* `360.0` で一回転
* `0` は完全に右
* 反時計回り
* `90` は真上
* `180` は左
* `270` は真下

例：

```text
移動角度 90 200 60
```

上へ200pxを60フレーム掛けて移動。

---

### 待つ

```text
待つ F
```

`F` フレーム間、その場で待機する。

例：

```text
待つ 30
```

30フレーム待機。

---

### 画像

```text
画像 <画像名>
```

敵画像を設定する。

指定された画像名は、以下のファイルへ解決される。

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\画像\<画像名>.png
```

例：

```text
画像 ザコ敵
```

↓

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\画像\ザコ敵.png
```

---

### 半径

```text
半径 R
```

敵の当たり判定半径を設定する。

例：

```text
半径 10
```
