# 敵キャラクタ制御スクリプト仕様

## 概要

KEnemy は、Annihilation における敵キャラクタの動作を記述するためのスクリプト言語である。

主に以下を制御する。

* 敵の移動
* 敵の画像
* 敵の当たり判定
* 敵の体力
* 敵・弾の生成

KEnemy は手続き型言語であり、以下をサポートする。

* 逐次実行
* 条件分岐
* 繰り返し
* 関数呼び出し
* 変数
* 数式

---

## スクリプトファイル

### 配置場所

```text
C:\Dev\Annihilation\Game\Storage\KEnemy
```

### ファイル名

```text
<関数名>.txt
```

例：

```text
ザコ敵.txt
放射弾.txt
ボス第一形態.txt
```

1ファイルが1関数に対応する。

---

## 文字コード

関数ファイルの文字コードは、以下のどちらかとする。

* UTF-8 with BOM
* Shift_JIS (CP932 / codepage 932)

改行コードは CRLF / LF のどちらでもよい。

---

## コメント

`#` で始まる行はコメント行として扱われる。

コメント行は実行されない。

例：

```text
# これはコメント

画像 敵小型

# 体力設定
HP = 10
```

コメントは行頭にのみ記述可能。

現時点では、行末コメントはサポートしない。

以下は未定義動作とする。

```text
HP = 10 # コメント この書き方は禁止！
```

---

## 実行モデル

KEnemy スクリプトは、上から順番に逐次実行される。

以下の命令は時間を消費する。

* `移動相対`
* `移動絶対`
* `移動角度`
* `待つ`

これらの命令は、完了するまで次の行を実行しない。

例：

```text
移動絶対 700 200 60
待つ 30
```

上記は：

1. 60フレーム掛けて移動
2. 30フレーム待機

の順で実行される。

---

## スクリプトの開始

ゲーム側から敵が生成されると、指定された関数名のスクリプトが実行される。

関数名は以下のファイルへ解決される。

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\<関数名>.txt
```

例：

```text
関数名: ザコ敵
```

↓

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\ザコ敵.txt
```

生成位置は `X`, `Y` に設定された状態で開始される。

開始時に引数が渡された場合は、`P1`, `P2`, `P3`, ... として参照できる。

---

## 変数

### 型

すべての変数は `double` 型である。

整数・浮動小数の区別は存在しない。

### 変数宣言

不要。

初回代入時に自動生成される。

例：

```text
I = 0
ANGLE = 90
```

### 変数名

以下の正規表現に一致する必要がある。

```regex
[A-Z][A-Z0-9]*
```

例：

```text
X
Y
HP
I
ANGLE
TARGETX
P1
P2
```

小文字は使用できない。

---

## 標準変数

各敵キャラクタは以下の変数を持つ。

| 変数名  | 意味    |
| ---- | ----- |
| `X`  | 現在X座標 |
| `Y`  | 現在Y座標 |
| `HP` | 現在体力  |

### X, Y

敵生成時に設定される。

座標系：

* バトルフィールド左上が `(0, 0)`
* X増加方向が右
* Y増加方向が下
* バトルフィールドのサイズは (456 x 532)

### HP

初期値は `1`。

敵弾など、プレイヤーが破壊できない敵は 0 にすること。

例：

```text
HP = 10
```

---

## 関数

### 関数定義

関数定義はファイル名によって行われる。

```text
放射弾.txt
```

↓

```text
放射弾
```

という関数になる。

ファイル内に関数名や引数名を宣言する構文は存在しない。

---

## 関数呼び出し

### 呼び出し形式

```text
<関数名> <引数1> <引数2> ...
```

例：

```text
放射弾 5 180
```

---

### 関数呼び出し時の変数

関数へ入る際、呼び出し前の通常変数は退避される。

呼び出された関数内で参照できる変数は、以下のみである。

* `X`
* `Y`
* `HP`
* `P1`, `P2`, `P3`, ...（引数が存在する場合）

`X`, `Y`, `HP` は特殊な変数であり、常にどの関数からでも参照・変更できる。

関数を終了すると、呼び出し前に退避された通常変数が復帰する。

関数内で作成・変更された通常変数は、関数終了時に忘却される。

---

## 関数の引数

関数呼び出し時に渡された引数は、呼び出された関数側で以下の変数に順番に格納される。

| 引数   | 格納先             |
| ---- | --------------- |
| 第1引数 | `P1`            |
| 第2引数 | `P2`            |
| 第3引数 | `P3`            |
| 第4引数 | `P4`            |
| ...  | ... |

例：

呼び出し側：

```text
放射弾 5 180
```

`放射弾.txt` 側：

```text
COUNT = P1
SPEED = P2
```

つまり、関数側で任意の名前にしたい場合は、冒頭で代入して受け取る。

```text
COUNT = P1
SPEED = P2
ANGLE_OFFSET = P3
```

## 数式

代入式、`もし` の条件、`繰り返し` の条件で使用できる。

命令や関数呼び出しの引数には、空白で区切られた1つの値を渡す。

複雑な計算結果を命令や関数に渡したい場合は、先に変数へ代入する。

例：

```text
TARGETX = X + 100
移動絶対 TARGETX Y 60
```

### 演算子

以下をサポートする。

```text
+
-
*
/
%
```

例：

```text
R = 360 / COUNT * I
X2 = X + 100
```

### 比較演算子

```text
==
!=
<
>
<=
>=
```

### 代入式

```text
<変数名> = <式>
```

左辺には代入先の変数名を書く。

右辺には数値、変数名、演算子、括弧を書ける。

例：

```text
A = B + 123
R = 360 / COUNT * I
```

未定義の変数を参照した場合、その値は `0` として扱われる。

### 値

以下を使用できる。

* 変数名
* 数値リテラル

例：

```text
A
10
3.5
```

### 空白

演算子や括弧は、空白で区切って書く。

例：

```text
A = ( B + 2 ) * 3
```

以下のように詰めて書かない。

```text
A=(B+2)*3
```

### 優先順位

* `(` と `)` による優先順位指定に対応する。
* `*`, `/`, `%` は `+`, `-` より優先される。
* 同じ優先順位の演算子は左から順に評価される。

### 例

```text
A = B + 123
```

`B` が `10` の場合、`A` は `133` になる。

```text
A = ( B + 2 ) * 3
```

`B` が `10` の場合、`A` は `36` になる。

```text
A = B % 4
```

`B` が `10` の場合、`A` は `2` になる。

## 制御構文

### もし

```text
もし 条件

    ...

もし終了
```

例：

```text
もし HP <= 0

    待つ 60

もし終了
```

### 繰り返し

```text
繰り返し 条件

    ...

繰り返し終了
```

条件が真の間、繰り返す。

例：

```text
I = 0

繰り返し I < 5

    I = I + 1

繰り返し終了
```

## 組み込み命令

以下は、スクリプト内であらかじめ使用できる命令である。

計算式を使いたい場合は、事前に変数へ代入してから指定する。

---

### 発射

```text
発射 <関数名> <引数1> <引数2> ...
```

敵キャラクタを生成する。

生成された敵は、指定された関数を実行する。

生成位置は呼び出し元の敵の現在位置となる。

`<関数名>` は文字列として扱われる。

`<引数>` には数値または変数名を指定する。

例：

```text
発射 弾 90 180
```

このとき、`弾.txt` 側では以下のように受け取る。

```text
R = P1
SPEED = P2
```

---

### 移動相対

```text
移動相対 X Y F
```

現在位置から相対的に `(X, Y)` だけ、`F` フレーム掛けて移動する。

`X`, `Y`, `F` には数値または変数名を指定する。

例：

```text
移動相対 100 0 60
```

右へ100pxを60フレーム掛けて移動。

---

### 移動絶対

```text
移動絶対 X Y F
```

絶対座標 `(X, Y)` へ `F` フレーム掛けて移動する。

`X`, `Y`, `F` には数値または変数名を指定する。

例：

```text
移動絶対 640 200 120
```

---

### 移動角度

```text
移動角度 R D F
```

角度 `R` 方向へ、距離 `D` を `F` フレーム掛けて移動する。

`R`, `D`, `F` には数値または変数名を指定する。

角度仕様：

* `360.0` で一回転
* `0` は完全に右
* 反時計回り
* `90` は真上
* `180` は左
* `270` は真下

例：

```text
移動角度 90 200 60
```

上へ200pxを60フレーム掛けて移動。

---

### 待つ

```text
待つ F
```

`F` フレーム間、その場で待機する。

`F` には数値または変数名を指定する。

例：

```text
待つ 30
```

30フレーム待機。

---

### 画像

```text
画像 <画像名>
```

敵画像を設定する。

指定された画像名は、以下のファイルへ解決される。

`<画像名>` は文字列として扱われる。

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\画像\<画像名>.png
```

例：

```text
画像 ザコ敵
```

↓

```text
C:\Dev\Annihilation\Game\Storage\KEnemy\画像\ザコ敵.png
```

---

### 半径

```text
半径 R
```

敵の当たり判定半径を設定する。

`R` には数値を指定する。

例：

```text
半径 10
```

---
